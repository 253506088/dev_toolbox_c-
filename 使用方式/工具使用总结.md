## 🚀 快速使用指南

### 日常开发

```powershell
# 运行应用
.\dev.ps1

# 热重载开发（推荐）
.\dev.ps1 watch

# 运行测试
.\dev.ps1 test

# 清理项目
.\dev.ps1 clean
```

### 发布打包

```powershell
# 标准发布（x64 单文件免安装版）
.\publish.ps1 -Version "1.0.0"

# 包含 ARM64 版本
.\publish.ps1 -Version "1.0.0" -IncludeArm64

# 跳过测试快速发布
.\publish.ps1 -Version "1.0.0" -SkipTests
```

### 性能测试

```powershell
# 运行完整性能测试
.\performance-test.ps1

# 创建测试文件（10万个文件 + 1000张图片）
.\performance-test.ps1 -CreateTestFiles

# 自定义数量
.\performance-test.ps1 -CreateTestFiles -FileCount 50000 -ImageCount 500

# 清理测试文件
.\performance-test.ps1 -CleanupTestFiles
```

## 📋 dev.ps1 功能

这是日常开发的主要工具，支持以下命令：

| 命令 | 功能 | 使用场景 |
|------|------|---------|
| `run` | 运行应用（默认） | 日常测试 |
| `watch` | 热重载模式 | 开发调试（推荐） |
| `test` | 运行单元测试 | 功能验证 |
| `build` | 构建项目 | 检查编译错误 |
| `clean` | 清理项目 | 解决构建问题 |
| `help` | 显示帮助 | 查看所有命令 |

**特色功能：**
- ✅ 自动清理旧日志（7天前）
- ✅ 彩色输出，易于阅读
- ✅ 自动创建测试项目（首次运行 test）
- ✅ 热重载支持（watch 命令）

## 📦 publish.ps1 功能

专业的发布打包工具，功能完整：

**主要功能：**
1. ✅ 环境检查（.NET SDK 版本）
2. ✅ 清理旧的构建产物
3. ✅ 可选的单元测试运行
4. ✅ 发布 x64 和 ARM64 版本
5. ✅ 自动复制 FFmpeg
6. ✅ 生成 README.txt
7. ✅ 创建 ZIP 压缩包
8. ✅ 生成 SHA256 校验和
9. ✅ 详细的进度显示

**参数：**
- `-Version` - 版本号（必需）
- `-SkipTests` - 跳过测试
- `-IncludeArm64` - 包含 ARM64 版本

**输出文件：**
- `DevToolbox-v{Version}-win-x64.zip` - x64 压缩包
- `DevToolbox-v{Version}-win-arm64.zip` - ARM64 压缩包（可选）
- `DevToolbox-v{Version}-checksums.txt` - SHA256 校验和
- `publish/win-x64/` - x64 发布目录
- `publish/win-arm64/` - ARM64 发布目录（可选）

## 🧪 performance-test.ps1 功能

全面的性能测试工具：

**测试项目：**
1. ✅ 应用启动时间（冷启动）
2. ✅ 内存占用（初始 + 稳定）
3. ✅ 完整构建时间
4. ✅ 增量构建时间
5. ✅ 发布包大小（单文件 + 压缩）

**测试数据生成：**

- 生成测试图片（默认 1000 张，需要 ImageMagick）

**性能目标：**
- 启动时间 < 5 秒
- 内存占用 < 300 MB
- 完整构建 < 30 秒
- 增量构建 < 10 秒
- 压缩包大小 < 100 MB


## ⚠️ 注意事项

### dev.ps1
- 首次运行 `test` 会自动创建测试项目
- `clean` 会删除所有 bin、obj 和 logs 目录
- `watch` 模式下修改 C# 文件需要手动保存触发重载

### publish.ps1
- 默认会运行测试，使用 `-SkipTests` 跳过
- 需要手动复制 FFmpeg 到 `assets/ffmpeg/`
- ARM64 版本需要额外指定 `-IncludeArm64`
- 生成的校验和文件用于用户验证下载完整性

### performance-test.ps1
- 创建 10 万个文件需要较长时间（取决于磁盘速度）
- 图片生成需要安装 ImageMagick
- 测试会短暂启动应用，确保没有其他实例运行
- 测试目录位于 `D:\DevToolbox_Test*`，可自行修改脚本
