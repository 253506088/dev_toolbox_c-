## 开发测试

### 1. 调试运行

```powershell
# 方式一：使用 dotnet run
cd D:\code\my\1\dev_toolbox\dev_toolbox_c#
dotnet run --project src/DevToolbox.App

# 方式二：使用 Visual Studio
# 打开 DevToolbox.slnx，按 F5 启动调试

# 方式三：使用 Rider
# 打开解决方案，选择 DevToolbox.App 项目，点击运行
```

### 2. 热重载（Hot Reload）

.NET 10 支持 WPF 热重载，在调试状态下修改 XAML 文件会自动刷新界面。

```powershell
# 启用热重载调试
dotnet watch run --project src/DevToolbox.App
```

### 3. 查看日志

日志文件位置：`logs/devtoolbox-YYYYMMDD.log`

```powershell
# 实时查看日志
Get-Content logs/devtoolbox-$(Get-Date -Format "yyyyMMdd").log -Wait
```

## 单元测试

### 1. 创建测试项目

```powershell
cd D:\code\my\1\dev_toolbox\dev_toolbox_c#\tests
dotnet new xunit -n DevToolbox.Tests -f net10.0
cd ..
dotnet sln add tests/DevToolbox.Tests/DevToolbox.Tests.csproj
```

### 2. 添加测试依赖

```powershell
cd tests/DevToolbox.Tests
dotnet add reference ../../src/DevToolbox.Core/DevToolbox.Core.csproj
dotnet add reference ../../src/DevToolbox.Infrastructure/DevToolbox.Infrastructure.csproj
dotnet add package Moq
dotnet add package FluentAssertions
```

### 3. 运行测试

```powershell
# 运行所有测试
dotnet test

# 运行指定测试类
dotnet test --filter "ClassName=StorageServiceTests"

# 运行测试并生成覆盖率报告
dotnet test --collect:"XPlat Code Coverage"

# 详细输出
dotnet test --logger "console;verbosity=detailed"
```

### 4. 测试示例

```csharp
// tests/DevToolbox.Tests/StorageServiceTests.cs
using Xunit;
using FluentAssertions;

namespace DevToolbox.Tests;

public class StorageServiceTests
{
    [Fact]
    public async Task InitializeAsync_ShouldCreateDatabase()
    {
        // Arrange
        var service = new StorageService();
        
        // Act
        await service.InitializeAsync();
        
        // Assert
        File.Exists("devtoolbox.db").Should().BeTrue();
    }
}
```

## 发布打包

### 1. 发布配置

#### 单文件免安装版（推荐）

```powershell
# Windows x64 单文件发布
dotnet publish src/DevToolbox.App -c Release -r win-x64 `
  --self-contained `
  -p:PublishSingleFile=true `
  -p:IncludeNativeLibrariesForSelfExtract=true `
  -p:EnableCompressionInSingleFile=true `
  -o publish/win-x64
```

#### 依赖框架版（体积更小）

```powershell
# 需要用户安装 .NET 10 运行时
dotnet publish src/DevToolbox.App -c Release -r win-x64 `
  --no-self-contained `
  -p:PublishSingleFile=true `
  -o publish/win-x64-framework
```

#### ARM64 版本

```powershell
# Windows ARM64 发布
dotnet publish src/DevToolbox.App -c Release -r win-arm64 `
  --self-contained `
  -p:PublishSingleFile=true `
  -o publish/win-arm64
```

### 2. 发布输出说明

发布后的文件结构：

```
publish/win-x64/
├── DevToolbox.App.exe          # 主程序
├── ffmpeg.exe                  # FFmpeg（需要手动复制）
└── logs/                       # 日志目录（运行时创建）
```

### 3. 添加 FFmpeg

```powershell
# 下载 FFmpeg 并复制到发布目录
# 从 https://ffmpeg.org/download.html 下载 Windows 版本
Copy-Item "path\to\ffmpeg.exe" "publish/win-x64/"
```

### 4. 打包成压缩包

```powershell
# 使用 PowerShell 创建 ZIP
Compress-Archive -Path "publish/win-x64/*" `
  -DestinationPath "DevToolbox-v1.0.0-win-x64.zip"

# 或使用 7-Zip（压缩率更高）
7z a -tzip DevToolbox-v1.0.0-win-x64.zip publish/win-x64/*
```

### 5. 发布优化选项

创建 `src/DevToolbox.App/DevToolbox.App.csproj` 添加发布配置：

```xml
<PropertyGroup Condition="'$(Configuration)' == 'Release'">
  <!-- 启用 AOT 编译优化 -->
  <PublishReadyToRun>true</PublishReadyToRun>
  
  <!-- 启用修剪（减小体积） -->
  <PublishTrimmed>true</PublishTrimmed>
  <TrimMode>link</TrimMode>
  
  <!-- 调试符号选项 -->
  <DebugType>none</DebugType>
  <DebugSymbols>false</DebugSymbols>
  
  <!-- 启用压缩 -->
  <EnableCompressionInSingleFile>true</EnableCompressionInSingleFile>
</PropertyGroup>
```

⚠️ **注意**：启用 `PublishTrimmed` 可能导致反射功能失效，需要谨慎测试。

### 6. 快速发布脚本

创建 `publish.ps1` 脚本：

```powershell
# publish.ps1
param(
    [string]$Version = "1.0.0"
)

$ErrorActionPreference = "Stop"

Write-Host "开始发布 DevToolbox v$Version..." -ForegroundColor Green

# 清理旧的发布文件
if (Test-Path "publish") {
    Remove-Item "publish" -Recurse -Force
}

# 发布 x64 版本
Write-Host "正在发布 x64 版本..." -ForegroundColor Cyan
dotnet publish src/DevToolbox.App -c Release -r win-x64 `
  --self-contained `
  -p:PublishSingleFile=true `
  -p:IncludeNativeLibrariesForSelfExtract=true `
  -p:EnableCompressionInSingleFile=true `
  -o publish/win-x64

# 复制 FFmpeg（如果存在）
if (Test-Path "assets/ffmpeg/ffmpeg.exe") {
    Copy-Item "assets/ffmpeg/ffmpeg.exe" "publish/win-x64/"
    Write-Host "已复制 FFmpeg" -ForegroundColor Green
}

# 创建 ZIP 包
Write-Host "正在创建压缩包..." -ForegroundColor Cyan
Compress-Archive -Path "publish/win-x64/*" `
  -DestinationPath "DevToolbox-v$Version-win-x64.zip" `
  -Force

# 显示结果
$size = (Get-Item "DevToolbox-v$Version-win-x64.zip").Length / 1MB
Write-Host "发布完成！" -ForegroundColor Green
Write-Host "文件：DevToolbox-v$Version-win-x64.zip" -ForegroundColor Yellow
Write-Host "大小：$([Math]::Round($size, 2)) MB" -ForegroundColor Yellow
```

使用方法：

```powershell
# 发布指定版本
.\publish.ps1 -Version "1.0.0"
```

## 常见问题

### 1. 发布后双击无法运行

**原因**：缺少 .NET 10 运行时（仅限非 self-contained 版本）

**解决**：
```powershell
# 下载并安装 .NET 10 Desktop Runtime
# https://dotnet.microsoft.com/download/dotnet/10.0
```

### 2. 发布体积过大

**原因**：包含了所有运行时和依赖

**优化方法**：
```powershell
# 1. 使用依赖框架的发布方式
dotnet publish --no-self-contained

# 2. 启用修剪（需要测试）
dotnet publish -p:PublishTrimmed=true

# 3. 压缩可执行文件
dotnet publish -p:EnableCompressionInSingleFile=true
```

### 3. 日志文件找不到

**原因**：相对路径问题

**解决**：修改日志配置使用绝对路径
```csharp
var logPath = Path.Combine(
    AppDomain.CurrentDomain.BaseDirectory, 
    "logs", 
    "devtoolbox-.log"
);
```

### 4. SQLite 数据库权限错误

**原因**：发布目录没有写权限

**解决**：将数据库放在用户目录
```csharp
var dbPath = Path.Combine(
    Environment.GetFolderPath(Environment.SpecialFolder.LocalApplicationData),
    "DevToolbox",
    "devtoolbox.db"
);
```

### 5. FFmpeg 找不到

**原因**：ffmpeg.exe 不在搜索路径

**解决**：
```csharp
// 在代码中指定 FFmpeg 路径
var ffmpegPath = Path.Combine(
    AppDomain.CurrentDomain.BaseDirectory,
    "ffmpeg.exe"
);
```

### 6. 发布后图标丢失

**解决**：在项目文件中指定图标

```xml
<PropertyGroup>
  <ApplicationIcon>icon.ico</ApplicationIcon>
</PropertyGroup>
```
